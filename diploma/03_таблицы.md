## Таблица 1.3 — Сравнительный анализ СУБД для прототипа

| Параметр | SQLite | MySQL | PostgreSQL |
|---|---|---|---|
| Лицензия | Public Domain | GPL/коммерч. | PostgreSQL License |
| Установка | Не требуется (файл) | Сервер + конфиг | Сервер + конфиг |
| Подходит для прототипа | Да | Избыточно | Избыточно |
| Поддержка JSON | Частичная | Полная | Полная |
| WAL-режим | Да | – | Да |
| Транзакции | ACID | ACID | ACID |
| Размер минимум | < 1 МБ | ~ 200 МБ | ~ 100 МБ |
| Подходит для до 100 пользователей | Да | Да | Да |
| Целевое решение | **Прототип** | Промышленная | Промышленная с PostGIS/JSONB |

**Вывод:** SQLite оптимален для прототипа бакалаврского уровня — нулевая стоимость администрирования, полная поддержка ACID, возможность миграции на PostgreSQL в production без изменения слоя доступа к БД (PDO).

## Таблица 2.2 — Структура таблиц БД

| Таблица | Поля | Назначение |
|---|---|---|
| users | id, role, full_name, phone, email, password_hash, totp_secret, totp_enabled, created_at | Пользователи всех ролей |
| patients | id, user_id, full_name, birth_date, normal_systolic, normal_diastolic, created_at | Профиль пациента и его норма |
| access | id, patient_id, user_id, is_active, granted_at | Привязка родственника/врача к пациенту |
| medicines | id, patient_id, name, dose, times, is_active, created_by, created_at | Назначенные лекарства |
| intakes | id, medicine_id, patient_id, scheduled_at, status, taken_at, skip_reason, comment | Конкретные приёмы лекарства |
| records | id, patient_id, created_by, recorded_at, systolic, diastolic, pulse, note | Записи дневника АД |
| events | id, patient_id, type, severity, message, status, created_at, resolved_by | Сгенерированные системой события |
| comments | id, patient_id, author_id, type, visibility, content, created_at | Сообщения и рекомендации |
| audit_log | id, user_id, action, entity, entity_id, ip, ts | Журнал действий пользователей |

**Итого:** 9 сущностей, 3НФ, foreign keys включены через `PRAGMA foreign_keys = ON`.
